<h2>Introduction&nbsp; </h2>With the introduction of Visual Studio 2017, the format used for csproj was updated including several advantages. This new format will be used when you create a new .Net Core project or .Net Standard library. In spite of the fact that VS2017 continues supporting old csproj format, you can migrate your old projects to SDK format. As a result, you will have the following benefits:<br /><br /><ul><li><b>Cleaner csproj files.</b> With new format, a lot of things can be omitted, so your csproj files will be cleaner. </li><li><b>Files are included using a wildcard</b>, instead of including every single file.&nbsp;</li><li>Solution project <b>references are simplier</b>.</li><li><b>NuGet reference packages as <a href="https://docs.microsoft.com/en-us/nuget/consume-packages/package-references-in-project-files">PackageReferences</a></b>, so you won't need packages.config and you will be able to use <a href="https://docs.microsoft.com/en-us/nuget/reference/package-versioning#version-ranges-and-wildcards">wildcard in your references</a> (for example, 1.0.*).</li><li><b>NuGet package definition as a part of the project file.</b> You won't need more nuspec files.</li><li><b>Definition of AssemblyInfo attributes in csproj</b>, you won't need more AssemblyInfo.cs files. In addition, you will be able to use features such as <b>Directory.Build.props</b> to centralize this info. Go to this link for more info: <a href="https://www.albertocorrales.com/2018/11/maintaining-assemblyinfo-for-multiple.html">https://www.albertocorrales.com/2018/11/maintaining-assemblyinfo-for-multiple.html</a></li><li><b>Your project will be ready to use .Net Standard 2</b> as a target if you want to do it in the future.</li></ul><h2>Migrate your csproj to SDK format</h2><h3>1) Migrating from Visual Studio (only PackageReferences) </h3>From Visual Studio, there is not an automatic way to convert your projects to SDK format. The only thing VS allow you to do form IDE is converting NuGet references to PackageReference format. To do it, just select "references" or "packages.config" and click on "Migration packages.config to PackageReferences".<br /><br /><div class="separator" style="clear: both; text-align: center;"><a href="https://3.bp.blogspot.com/-qQ9o2-bpsgY/XARLREXj6UI/AAAAAAAAFEo/0Xvc9nmR21sG630RASORsMpD8NUonyPjgCLcBGAs/s1600/1.png" imageanchor="1" style="margin-left: 1em; margin-right: 1em;"><img border="0" data-original-height="398" data-original-width="474" height="335" src="https://3.bp.blogspot.com/-qQ9o2-bpsgY/XARLREXj6UI/AAAAAAAAFEo/0Xvc9nmR21sG630RASORsMpD8NUonyPjgCLcBGAs/s400/1.png" width="400" /></a></div><div class="separator" style="clear: both; text-align: center;"><br /></div><br />In order to migrate ASP.NET projects, you won't see the previous menu, so you will have to do the <a href="https://github.com/NuGet/docs.microsoft.com-nuget/issues/860#issuecomment-409207305">following workaround</a>: <br /><ul><li>Open the csproj</li><li>Replace the ProjectGuid by <i>&lt;ProjectGuid&gt;{7C796B6B-86B5-4C57-ADAA-12CF1FECDA71}&lt;/ProjectGuid&gt; </i>and remove <i>ProjectTypeGuids</i></li><li>Now right click on reference, you should be able to migrate</li><li>Put back the previous ProjectType and ProjectTypeGuids</li></ul><h3>2) SDK format manual migration</h3>In order to convert your csproj, the default method is manual conversion. To do it, the easiest way is starting your csproj from the scratch. To know all the things you can remove, you can follow this tutorial, which explains the conversion process: <a href="https://natemcmaster.com/blog/2017/03/09/vs2015-to-vs2017-upgrade/">https://natemcmaster.com/blog/2017/03/09/vs2015-to-vs2017-upgrade/</a><br /><h3>3) SDK format migration using Tool CsprojToVs2017</h3>Manual mode can be useful if you have few project, but it is difficult to scale if you have a lot of projects to convert. Luckily, there is a GitHub project named CsprojToVs2017, which can be very handy to convert multiple projects automaticly: <a href="https://github.com/hvanbakel/CsprojToVs2017">https://github.com/hvanbakel/CsprojToVs2017</a><br /><br />This tool is very helpful, but before migration, you will see the resulting conversion it not as clearer as it should, you can finish the migration manually by removing unnecessary parts in csproj.<br /><h2>Migration Example</h2><h3>Old csproj </h3>Here we have an example of a migration for a project.
